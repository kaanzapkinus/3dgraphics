# Final Project — Procedural Solar System (Blender + Python)

A proposal / roadmap for the 3D Graphics final project, using **Blender's
Python API (`bpy`)** as required for Lesson 5.
Waiting for the professor's approval before starting implementation.

## Idea

A single Python script (`solar_system.py`) that is executed **inside
Blender** and, from nothing, programmatically builds a stylised solar
system: the Sun, several planets with proper relative sizes and orbit
radii, materials and lighting, an animated orbit, and a cinematic
camera. The script then renders the result.

Nothing in the scene is built by hand in the Blender UI — the entire
scene is generated by code. That is the whole point of `bpy`: showing
that 3D content authoring can be **scripted** instead of clicked.

## Why this project

It directly answers the Lesson 5 brief — Blender (the modelling /
authoring software) combined with Python (`bpy`) for procedural content
generation. It also reuses concepts from every earlier lesson:

| Lesson | Where it shows up |
|--------|-------------------|
| Lesson 2 | Translations, rotations, scaling, hierarchical animation (orbits) |
| Lesson 3 | Materials, emission shader for the Sun, point light, animation keyframes |
| Lesson 4 | Mesh primitives (UV spheres), procedural geometry |
| Lesson 5 | Blender + Python scripting (`bpy`), Blender as the authoring tool |

## Planned features

Core (script generates the following on each run):
- Clear-scene preamble (remove any leftover objects)
- The Sun — UV sphere at the origin with an emission shader (glow)
- A configurable list of planets, each:
  - Created with its own UV sphere
  - Scaled to its relative size
  - Placed at its orbital radius
  - Given a distinct procedural material (color, roughness)
  - Animated to orbit the Sun with its own period (keyframes / drivers)
- A point light at the Sun position, providing the only scene light
- A camera placed and oriented for a cinematic three-quarter view
- Optional rotating camera path for a turntable feel
- Render settings (resolution, sample count, output path) set in code
- One-line call at the bottom of the script that renders the still image

Optional / stretch goals:
- Star field background (HDRI or particle system from a script)
- Asteroid belt generated procedurally (instances on a torus)
- Saturn-style ring around one planet
- Render an animated turntable as an `.mp4` / image sequence instead
  of a still
- Expose all the scene parameters as constants at the top of the file
  so the same script can produce different solar systems by editing
  five numbers

## Tech stack

- Blender 4.x
- Python 3.x (already bundled with Blender)
- `bpy` — Blender's built-in Python API
- No external pip packages required

## Implementation steps

1. **Scene reset helper** — function that clears the default cube /
   camera / light so the script is idempotent.
2. **Material builder** — small helper that creates a node-based
   material with a given base color, roughness, and optional emission.
3. **Sun + planets** — driven by a config list at the top of the file
   (`[(name, radius, distance, color, period), ...]`), looped to add
   each body.
4. **Animation** — insert keyframes on the planet's rotation around
   the origin so it orbits over the chosen period.
5. **Lighting + camera** — point light at the Sun's position, perspective
   camera with a fixed target.
6. **Render** — set output path, resolution, engine (Cycles or Eevee),
   and call `bpy.ops.render.render(write_still=True)`.
7. **README + sample render** — commit the script, a short README,
   and one PNG of the result so it can be reviewed without running
   Blender locally.

## How it will be delivered

Running the project locally:

```text
1. Open Blender (version 4.x)
2. Switch to the Scripting workspace
3. Open solar_system.py
4. Click "Run Script"
```

…and the rendered image is written next to the script.
